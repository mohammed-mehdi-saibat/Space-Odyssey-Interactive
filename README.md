# SpaceX Exploration Website

## Contexte du Projet

Ce projet consiste à transformer un site web statique (One-Page) en un site multi-pages interconnectées. Le thème central du site est l'exploration spatiale, mettant en avant SpaceX, ses fusées, et ses missions emblématiques. L'objectif est de présenter de manière immersive et attrayante le contenu scientifique, technologique et visuel lié à l'exploration spatiale, avec une attention particulière aux missions de SpaceX et à ses innovations.

## Objectif du Projet

L'objectif est de créer un site web structuré avec plusieurs pages interconnectées, en développant les sections suivantes :

- **Page d'accueil (Home)** : Introduction au thème “Space Discovery” avec des visuels immersifs et engageants.
- **Page “Missions”** : Présentation des missions emblématiques de SpaceX comme Falcon 9, Starship, Crew Dragon, et Starlink.
- **Page “Technologies”** : Détails sur les innovations techniques de SpaceX, y compris la réutilisation des fusées, la propulsion et l’ingénierie avancée.
- **Page “Contact”** : Formulaire complet pour permettre aux utilisateurs de rejoindre la communauté des passionnés de SpaceX.

### Travail à Réaliser

#### Analyse et Création

1. **Analyse de la maquette One-Page** : Identifier les sections principales à séparer en pages multiples.
2. **Création des Pages Web** : Minimum quatre pages, interconnectées, avec une navigation fluide entre elles :
   - **Page d’accueil** : Introduction visuelle à l’exploration spatiale, avec une présentation générale.
   - **Page “Missions”** : Détail des missions célèbres de SpaceX.
   - **Page “Technologies”** : Mise en valeur des technologies utilisées par SpaceX, telles que la réutilisation de fusées et les systèmes de propulsion avancés.
   - **Page “Contact”** : Formulaire pour les utilisateurs, afin qu'ils puissent rejoindre la communauté.
3. **Personnalisation et Amélioration** :

   - **Design créatif** : Modifier les couleurs, typographies et le style visuel pour améliorer l’expérience utilisateur.
   - **Animations et interactions** : Ajouter des animations CSS ou des interactions JavaScript pour dynamiser l’interface.
   - **Enrichissement du contenu** : Ajouter des sections supplémentaires, comme “Space News” ou “Launch Schedule”, ou modifier les contenus existants (images, textes).

4. **Compatibilité et Responsivité** :

   - Assurer que le site soit responsive et compatible avec différents navigateurs.

5. **Hébergement sur GitHub Pages** :
   - Héberger la version finale du site sur GitHub Pages.
   - Organiser le dépôt GitHub avec des commits clairs et réguliers, montrant l’évolution du projet.

### Modalités d'Évaluation

- **Démonstration du site** : Montrer le site dans le navigateur, démontrer ses fonctionnalités et sa fluidité.
- **Explication du code** : Présenter et expliquer le code source (HTML/CSS) utilisé dans le projet.
- **Revue du dépôt GitHub** : Vérification de l’organisation du dépôt et de la régularité des commits.

### Livrables

- **Lien du dépôt GitHub** : Contenant le code source complet du projet.
- **Lien du site GitHub Pages** : Hébergement final du site web accessible publiquement.

### Critères de Performance

1. **Pages Interconnectées** : Le site doit comporter au minimum 4 pages interconnectées, avec une page “Contact”.
2. **Thème Identifiable** : Le thème “Astronomy / SpaceX” doit être clairement identifiable à travers le design et le contenu.
3. **Navigation Fluide** : La navigation entre les pages doit être intuitive et fluide.
4. **Design Moderne et Responsive** : Le design du site doit être cohérent, moderne et responsive sur différents appareils.
5. **Code Validé** : Le code HTML/CSS doit être valide, bien structuré et propre.
6. **Hébergement GitHub Pages Fonctionnel** : Le site doit être accessible via GitHub Pages sans erreurs.
